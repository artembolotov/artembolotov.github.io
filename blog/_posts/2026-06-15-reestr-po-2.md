[В&nbsp;прошлый раз](/blog/2025/05/28/reestr-po/) я&nbsp;пытался зарегистрировать TwinKey в&nbsp;реестре отечественного&nbsp;ПО и&nbsp;сдался. Форма была рассчитана явно не&nbsp;на&nbsp;инди-разработчика, ЭЦП нужна была как в&nbsp;шпионском кино, а&nbsp;внятных преимуществ никто так и&nbsp;не&nbsp;объяснил.

## Письмо счастья

Прошёл год. Утром пришло письмо:

> Здравствуйте, Болотов Артем Андреевич.
>
> Установлен статус «закрыто» для заявления о&nbsp;включении сведений о&nbsp;ПО.
>
> Наименование программного обеспечения: TwinKey 2FA Authenticator  
> Наименование правообладателя: Болотов Артем Андреевич  
> Номер заявления: 335974

И&nbsp;предложение оценить качество услуги. Услуга, напомню, заключалась в&nbsp;том, чтобы закрыть мою заявку и&nbsp;ничего не&nbsp;объяснить.

Значит, прошлая попытка всё-таки не&nbsp;просто зависла в&nbsp;воздухе, а&nbsp;официально завершилась. Хорошо&nbsp;&mdash; можно начинать заново с&nbsp;чистого листа, теперь умнее.

## Как я&nbsp;готовился

На&nbsp;этот раз перед заполнением формы я&nbsp;разобрался в&nbsp;деталях заранее&nbsp;&mdash; методом вопросов и&nbsp;ответов с&nbsp;ИИ, который уже проходил через эти формы с&nbsp;другими разработчиками. Ниже&nbsp;&mdash; самые неочевидные моменты, которые я&nbsp;выяснил.

## Как называть и&nbsp;куда класть

Первый вопрос&nbsp;&mdash; как написать название. Казалось бы, что тут думать. Но&nbsp;нет:

Название вносится без платформенных приставок. Не&nbsp;«TwinKey для iOS», а&nbsp;просто «TwinKey». Это не&nbsp;формальность&nbsp;&mdash; запись в&nbsp;реестре считается кроссплатформенной по&nbsp;умолчанию, и&nbsp;если когда-нибудь выйдет Android-версия, не&nbsp;придётся переименовывать. Функциональное описание («двухфакторная защита») идёт в&nbsp;отдельное поле, не&nbsp;в&nbsp;название.

**Класс ПО**&nbsp;&mdash; 03.01 «Средства защиты от&nbsp;несанкционированного доступа к&nbsp;информации». Стандартная категория для 2FA-приложений.

**Код ОКПД2**&nbsp;&mdash; 58.29.32 (прикладное ПО для загрузки) и&nbsp;62.01.29 (оригиналы ПО прочие). Соответствует цифровой дистрибуции без физических носителей.

## Галочка, которую нельзя ставить

Есть пункт «Защита конфиденциальной информации». Для приложения двухфакторной аутентификации это звучит как очевидное «да».

Не&nbsp;ставьте.

Эта галочка означает, что ПО является **средством защиты конфиденциальной информации** в&nbsp;юридическом смысле. А&nbsp;это требует сертификата ФСТЭК на&nbsp;само&nbsp;ПО и&nbsp;лицензии ФСТЭК у&nbsp;правообладателя на&nbsp;разработку СЗКИ по&nbsp;Постановлению №171. Это отдельный долгий и&nbsp;дорогой процесс, никак не&nbsp;связанный с&nbsp;обычным TOTP-аутентификатором.

TwinKey не&nbsp;шифрует чужие данные и&nbsp;не&nbsp;защищает конфиденциальные сведения в&nbsp;регуляторном смысле&nbsp;&mdash; он&nbsp;генерирует одноразовые коды. Галочка не&nbsp;ставится.

## GitHub&nbsp;&mdash; неожиданная проблема

Форма спрашивает «адрес нахождения технических средств хранения исходного текста ПО». И&nbsp;тут открытый код на&nbsp;GitHub создаёт конфликт: по&nbsp;требованиям реестра, хранение исходного кода должно быть на&nbsp;территории РФ.

Написать «GitHub» нельзя&nbsp;&mdash; это сервер в&nbsp;США.

Решение для разработчика-одиночки: указать **личный компьютер на&nbsp;территории РФ**. Сборка идёт локально через Xcode на&nbsp;Mac&nbsp;&mdash; это и&nbsp;есть «технические средства компиляции». Подтверждается гарантийным письмом от&nbsp;правообладателя, а&nbsp;не&nbsp;договором аренды сервера (который нужен только если хранение обеспечивает кто-то другой).

## Бесплатное приложение с&nbsp;донатами

Поле про стоимость ПО предлагает варианты: «открытая лицензия / безвозмездно» или «по&nbsp;тарифной политике». Они взаимоисключающие.

TwinKey бесплатен. Покупки внутри приложения&nbsp;&mdash; это чаевые, не&nbsp;оплата функциональности. Значит&nbsp;&mdash; первый вариант.

Про выплаты иностранным лицам: комиссия Apple и&nbsp;взносы за&nbsp;Developer Program&nbsp;&mdash; это плата за&nbsp;дистрибуцию и&nbsp;доступ к&nbsp;платформе, а&nbsp;не&nbsp;«выплаты по&nbsp;лицензионным договорам за&nbsp;разработку и&nbsp;адаптацию ПО». Для разработчика без внешних подрядчиков сумма в&nbsp;этом поле&nbsp;&mdash; ноль. Подтверждается гарантийным письмом в&nbsp;свободной форме.

## Экземпляр ПО&nbsp;&mdash; это не&nbsp;исходный код

Форма просит «ссылку на&nbsp;экземпляр ПО для экспертной проверки». Это ссылка на&nbsp;App&nbsp;Store&nbsp;&mdash; рабочее приложение, которое эксперт установит и&nbsp;запустит. Не&nbsp;на&nbsp;репозиторий с&nbsp;кодом.

## Четыре обязательных документа

Для экспертной проверки нужно приложить:

1. **Инструкция по&nbsp;установке.** По сути&nbsp;&mdash; ссылка на&nbsp;App&nbsp;Store и&nbsp;пояснение, что нужен iPhone.

2. **Описание функциональных характеристик.** Что умеет приложение.

3. **Описание поддержания жизненного цикла.** Кто поддерживает, как сообщают о&nbsp;багах, как выходят обновления.

4. **Инструкция по&nbsp;эксплуатации.** Здесь есть нюанс&nbsp;&mdash; нужно вложить тестовые QR-коды, по&nbsp;которым эксперт проверит работу:
   - QR в&nbsp;формате `otpauth://totp/...`&nbsp;&mdash; для проверки добавления одиночного аккаунта (через сканирование, загрузку фото или ручной ввод)
   - QR в&nbsp;формате `otpauth-migration://offline?data=...`&nbsp;&mdash; это формат экспорта Google&nbsp;Authenticator, для проверки массового импорта из&nbsp;конкурирующего приложения

## Один в&nbsp;поле воин&nbsp;&mdash; снова

Поле про техподдержку предлагает категорию «силами штатных сотрудников». Для инди-разработчика это читается как «у&nbsp;нас что, должен быть штат?».

Нет. Формулировка трактуется как «собственными силами правообладателя»&nbsp;&mdash; применима и&nbsp;к&nbsp;физлицу без юрлица и&nbsp;без сотрудников. Это и&nbsp;есть «штатный сотрудник» в&nbsp;единственном числе.

## Галочки про управление из-за рубежа

В&nbsp;конце&nbsp;&mdash; чекбоксы подтверждения. Один из&nbsp;них: «Не&nbsp;имеет принудительного обновления и&nbsp;управления из-за рубежа».

Звучит страшно, но&nbsp;смысл простой: кто контролирует код и&nbsp;обновления? Дистрибуция через App&nbsp;Store (американская витрина) не&nbsp;считается&nbsp;&mdash; важно, что именно я&nbsp;решаю, что входит в&nbsp;обновление и&nbsp;когда оно выходит. App&nbsp;Store&nbsp;&mdash; транспорт, не&nbsp;управление.

Ещё один чекбокс&nbsp;&mdash; «GUI на&nbsp;русском языке». Тут важно: реально зайти в&nbsp;приложение с&nbsp;русской локалью и&nbsp;убедиться, что всё переведено, прежде чем ставить галочку.

## Подписание

Когда форма заполнена, систем предлагает подписать заявление через КриптоПро&nbsp;&mdash; флешка, USB-A, драйверы из&nbsp;2007&nbsp;года, привет из&nbsp;прошлого.

Есть способ современнее: скачать PDF заявления, зайти на&nbsp;Госуслуги в&nbsp;раздел [«Подписание документов»](https://www.gosuslugi.ru/600373/1/form), загрузить файл и&nbsp;отправить себе на&nbsp;подпись в&nbsp;приложение Госключ. Никакой флешки, подпись квалифицированная.

Тут я&nbsp;споткнулся дважды.

**Ошибка первая:** подписал УНЭП вместо УКЭП. Это разные вещи&nbsp;&mdash; неквалифицированная и&nbsp;квалифицированная электронная подпись. Для реестра нужна именно квалифицированная (УКЭП). Пришлось переподписывать.

**Ошибка вторая:** после того, как подписал УКЭП, оказалось, что хэш PDF изменился. Это значит, что файл, который я&nbsp;подписал, уже не&nbsp;совпадает с&nbsp;текущей версией заявления в&nbsp;системе. Пришлось скачать свежий PDF заявления заново, отправить его на&nbsp;подпись, подписать ещё раз&nbsp;&mdash; только тогда заявление ушло.

Если коротко: скачивайте PDF непосредственно перед подписанием. Не&nbsp;оставляйте его лежать, пока разбираетесь с&nbsp;Госключем.

## Что дальше

Заявление отправлено. Теперь ждём решения оператора реестра.
